User (wzq) is extending NeoVerse (CVPR2026, 4D world model: VGGT/WorldMirror reconstructor + Wan2.1 diffusion control branch) to autonomous-driving surround cameras, producing drone-follow novel-view 4D video. Source clips: DreamForge 6-camera ring in `res/` (cam_front, front_left/right, back, back_left/right).

Key constraint: the released reconstructor's motion model asserts a single monotonic time axis (`forward_timestamp >= timestamp` in worldmirror rasterization.py), so 6 cameras cannot share one time sequence. Also WorldMirror has no bundle adjustment, so a joint static surround pass produces 6 disconnected curved "billboard" shells.

Settled method = **v4** in `fuse_surround_drone.py` (NOT Umeyama, NOT joint static pass). Per-camera 4D reconstruction → rigid-rig anchoring with hardcoded nuScenes 6-cam extrinsics (`NUSCENES_RIG`) → ground-plane RANSAC in ego frame for per-camera metric scale + leveling rotation (`level_and_scale`) → merge into ego frame (composite similarity `R_total=R_lev·R_ec·R_wc`) → render. Key flags: `--dump_ply`, `--max_range` (cull far field), `--freeze_time`, `--ground_fill`, `--car` (red wireframe box). v5 added `--splat_scale` but user reverted to v4 (preferred). Full write-up in `report.md` (v4 section). Residual artifacts (wedge seams between views, central hole, far-field curl) are fundamental to monocular 2.5D shells without the skipped diffusion repair.

**2026-06-21 new direction — "战争迷雾"渐进高斯世界 on REAL nuScenes:** goal = switch from DreamForge synthetic clips to real nuScenes (`/mnt/ssd1/jzl/datasets/nuscenes`), use GT map + camera intr/extr to FORCE-constrain the reconstructor, then accumulate a gapless surround world that reveals like fog-of-war as the ego drives. User's core hypothesis: VGGT/WorldMirror predicts cameras inaccurately → that's what causes the surround discontinuity → feed GT cameras to fix it. KEY mechanism discovered: WorldMirror.forward already accepts GT cameras as POW3R-style priors — `cond_flags=[depth,rays,camera]` + `views['camera_poses']` ([B,S,4,4] c2w OpenCV) + `views['camera_intrs']` ([B,S,3,3] K). `reconstructor` (ModelManager fetch) IS a WorldMirror instance, so no source edits needed. First script written: `fog_world_v0_nuscenes.py` (repo root) — loads ONE nuScenes sample's 6 cams, builds GT c2w in CAM_FRONT ego ref frame + K adjusted for center_crop, runs a single static joint pass, A/B via `--use_gt` (cond_flags=[0,1,1] vs baseline [0,0,0]), dumps PLY for SuperSplat seam comparison. Note: with priors the output splats live in the network's NORMALIZED frame (normalize_poses), so metric/map fusion is deferred to v1. Roadmap M1–M5 in chat (Umeyama GT scale, temporal accumulation + voxel fog-of-war fusion, static/dynamic split via GT boxes, map-guided ground fill).

**v1 written: `fog_world_v1_nuscenes.py`** — ego-forward "fog-of-war" accumulation. Walks T keyframes (`sample['next']`, `--time_stride`), per-timestep does the v0-style 6-cam GT-prior pass, then KEY mechanism: per-timestep `umeyama(predicted 6 cam centers → GT global cam centers)` recovers a metric similarity (s,R,t) that drops each timestep's splats into ONE metric GLOBAL frame (sidesteps the normalize_poses problem; well-posed since 6 cams around the car are non-collinear). Accumulates, optional `--max_range` cull around ego. Phase 3 renders an EGO forward-driving view rendering ONLY splats accumulated up-to-t (the reveal), with `--frames_per_step` camera interpolation between keyframes. Global frame z=up so no PCA needed. Reuses v4 `transform_gaussians_`/`cull_by_radius_`. Dumps full-accumulation PLY. Known v1 limits (for v2): overlapping per-timestep surfaces double up (no voxel dedup yet), dynamic objects ghost (no static/dynamic split yet).
